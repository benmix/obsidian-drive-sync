# 排障

## 1. 构建后插件未加载

检查项：

- 确认 `dist/` 下已生成插件构建产物。
- 确认插件包已经链接或复制到 `<Vault>/.obsidian/plugins/<plugin-id>/`。
- 确认 `manifest.json` 仍然与插件目录和预期插件 ID 保持一致。

建议操作：

1. 执行 `pnpm run build`。
2. 使用 `pnpm run link:obsidian -- --vault "/path/to/YourVault"` 重新链接插件，或手动复制构建产物。
3. 重新加载 Obsidian，并检查插件是否出现在社区插件列表中。

## 2. 缺少构建产物

检查项：

- 确认 `pnpm install` 已成功完成。
- 确认 TypeScript 和 Vite 的构建错误已被修复，而不是被忽略。
- 确认 `dist/` 中存在 `main.js`、`manifest.json`，以及适用时的 `styles.css`。

建议操作：

1. 执行 `pnpm run build`。
2. 先修复报出的 TypeScript 或打包错误。
3. 在排查运行时问题前，重新确认 `dist/` 输出是否完整。

## 3. 命令未显示

检查项：

- 确认命令注册仍在插件启动过程中执行。
- 确认 command ID 保持唯一且稳定。
- 确认插件确实成功加载，而不是在启动早期就失败。

建议操作：

1. 确认插件已在 Obsidian 中启用。
2. 在 Obsidian 开发者控制台中查看启动错误。
3. 确认最近重构后命令注册仍然发生。

## 4. 设置未持久化

检查项：

- 确认 `loadData()` 和 `saveData()` 已正确 `await`。
- 确认设置写入没有在同一条流程后续又被覆盖。
- 确认设置 UI 在状态变化后会重新渲染。

建议操作：

1. 保存一次设置变更。
2. 重载插件或重启 Obsidian。
3. 确认该值已被持久化并被正确读回。

## 5. 状态 UI 表现不一致

检查项：

- 确认 runtime state、持久化 state 和渲染后的状态 UI 使用的是同一套语义。
- 确认认证暂停、同步活动和错误展示都来自正确的 source of truth。
- 确认状态行为变化时，展示逻辑也同步更新。

建议操作：

1. 复现状态切换过程。
2. 对比 runtime state、持久化 state 和最终渲染的状态输出。
3. 在真正拥有该状态的边界修复问题，而不是只在 UI 层打补丁。

## 6. 共享代码中混入桌面专用行为

检查项：

- 确认该能力确实只面向桌面。
- 确认该行为没有泄漏到 provider 无关层或共享同步逻辑中。
- 确认 `manifest.json` 正确表达了平台支持范围。

建议操作：

1. 如果桌面专用行为已泄漏到共享模块中，将其移回 runtime 边界之后。
2. 重新检查 `manifest.json` 以及相关文档说明。
3. 在预期平台假设下重新验证受影响路径。
